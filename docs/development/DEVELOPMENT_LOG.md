# Development Log

## 문서 목적

이 문서는 `gpt-share-manager` 프로젝트의 개발 과정, 의사결정 배경, 문제 해결 과정을 기록한다.

`PROJECT_STATUS.md`가 현재 상태 중심 문서라면, 이 문서는 왜 그렇게 결정했는지를 남기는 기록 문서다.

## 2026-06-24 Phase F-0 기록

### 프로젝트 방향 확정

Flask 버전은 기존 Apps Script 운영 버전과 별도 프로젝트로 진행한다.

결정 이유:

```text
기존 Apps Script 버전은 실제 운영 안정성이 목적이다.
Flask 버전은 PostgreSQL, Docker Compose, OCI 배포, CSV 이력 관리, AI 협업 개발 과정을 학습하기 위한 별도 프로젝트다.
```

### AI 협업 개발 방식 확정

이 프로젝트는 AI가 전체 앱을 대신 만드는 방식으로 진행하지 않는다.

결정한 방향:

```text
사용자가 요구사항과 최종 반영 여부를 결정한다.
ChatGPT는 설계 보조, 리뷰, 오류 분석, 보안 검토를 담당한다.
Codex는 자동 구현자가 아니라 보조 개발자와 리뷰어로 사용한다.
코드는 사용자가 이해 가능한 단위로 제시하고 직접 적용한다.
```

### 점진적 구조화 결정

처음부터 완성형 아키텍처를 만들지 않고, 작동하는 코드에서 시작해 필요가 생기면 최소 범위로 구조화한다.

결정 이유:

```text
초기 학습 단계에서 복잡한 폴더 구조는 코드 흐름 파악을 방해한다.
파일이 너무 커지면 리뷰 자체를 하지 않게 된다.
각 Phase 종료 시 Phase Maintenance로 리뷰 가능한 상태를 유지한다.
```

### uv 채택

Python 의존성 관리는 `requirements.txt`가 아니라 `uv + pyproject.toml + uv.lock`을 사용한다.

결정 이유:

```text
재현 가능한 실행 환경을 만들 수 있다.
Docker/OCI 배포와 궁합이 좋다.
의존성 선언과 잠금 파일의 의미를 학생들에게 설명하기 좋다.
```

### 문서 배치 원칙 확정

루트에는 핵심 문서만 두고, 세부 문서는 `docs/` 아래에 역할별로 둔다.

결정 이유:

```text
루트 과밀을 방지한다.
새 작업자는 핵심 문서부터 읽을 수 있다.
기술 선택과 의사결정 기록을 ADR로 남길 수 있다.
```
### Git 작업 흐름과 커밋 메시지 원칙 확정

커밋 메시지는 `[Phase] type: 한국어 설명` 형식을 기본으로 사용한다.

결정 이유:

```text
Git 기록을 단순 저장 기록이 아니라 개발 과정과 판단의 기록으로 남긴다.
영어 type prefix로 변경 성격을 빠르게 구분한다.
한국어 설명으로 학교/수업/개인 학습 맥락에서 읽기 쉽게 한다.
Phase prefix로 프로젝트 진행 흐름을 추적한다.
Codex가 만든 변경도 사용자가 diff를 검토하고 승인한 뒤 커밋한다.
```
