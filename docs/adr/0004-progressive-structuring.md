# ADR 0004: 점진적 구조화와 Phase Maintenance 채택

## 상태

승인됨

## 맥락

처음부터 완성형 아키텍처를 만들면 초급~중급 학습자는 코드 흐름보다 폴더 구조에 압도될 수 있다. 반대로 단일 파일을 너무 오래 유지하면 리뷰를 포기하게 된다.

## 결정

작동하는 코드에서 시작하고, 불편함이 생기면 최소 범위로 구조화한다. 각 Phase 종료 시 Phase Maintenance를 수행한다.

## 대안

- 처음부터 완성형 아키텍처
- 마지막에 몰아서 리팩토링
- 점진적 구조화

## 결과

학습 흐름과 유지보수성을 동시에 확보할 수 있다.
