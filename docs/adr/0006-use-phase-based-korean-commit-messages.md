# ADR 0006: Phase 기반 한국어 커밋 메시지 사용

## 상태

Accepted

## 맥락

이 프로젝트는 단순 자동 생성물이 아니라, 사용자가 AI와 협업하면서 전체 개발 과정을 이해하고 통제하는 것을 목표로 한다.

따라서 Git 커밋 기록도 단순 저장 지점이 아니라 다음 역할을 가진다.

```text
작업 흐름 기록
Phase 진행 상황 기록
나중에 학생에게 보여줄 수 있는 개발 과정 자료
유지보수 시 변경 이유를 찾는 기준
```

일반적인 오픈소스 프로젝트에서는 영어 커밋 메시지를 많이 사용한다. 그러나 이 프로젝트는 한국어 기반 수업, 학교 업무 환경, 개인 학습/포트폴리오 목적이 강하다.

## 결정

커밋 메시지는 다음 형식을 기본으로 사용한다.

```text
[Phase] type: 한국어 설명
```

예시:

```text
[F-0] docs: 프로젝트 명세와 AI 협업 개발 원칙 정리
[F-1] build: uv 프로젝트 초기화
[F-1] feat: Flask 최소 앱 실행
[F-2] feat: SQLAlchemy User 모델 추가
```

`type`은 영어 prefix를 사용한다.

```text
docs
feat
fix
refactor
test
style
build
chore
```

설명은 한국어를 허용하고 권장한다.

## 대안

### 대안 1. 전체 영어 커밋 메시지

장점:

```text
오픈소스 관행과 잘 맞음
외부 개발자 협업에 유리함
```

채택하지 않은 이유:

```text
국내 수업/학교 업무 맥락에서는 읽는 속도가 떨어질 수 있음
학생들에게 개발 흐름을 설명하는 자료로는 한국어가 더 직접적임
```

### 대안 2. 전체 한국어 커밋 메시지

장점:

```text
읽기 쉬움
수업 자료로 적합함
```

채택하지 않은 이유:

```text
커밋 성격 분류가 흐려질 수 있음
GitHub, changelog, 자동화 도구 관행과의 연결성이 떨어질 수 있음
```

### 대안 3. prefix 없는 자유 형식

채택하지 않은 이유:

```text
기록이 누적될수록 검색과 분류가 어려워짐
AI/Codex와 협업할 때 커밋 성격을 명확히 전달하기 어려움
```

## 결과

긍정적 영향:

```text
커밋 로그만 봐도 Phase 흐름을 파악할 수 있다.
커밋 성격과 변경 내용을 동시에 읽기 쉽다.
학생들에게 Git 기록을 설명하기 좋다.
AI/Codex 작업 결과를 사람이 통제한 기록으로 남길 수 있다.
```

주의할 점:

```text
커밋 메시지가 너무 길어지지 않게 한다.
'수정', '작업함', '업데이트' 같은 모호한 메시지는 사용하지 않는다.
refactor와 feat를 섞지 않는다.
보안/권한 정책 변경은 메시지에 명확히 드러낸다.
```
